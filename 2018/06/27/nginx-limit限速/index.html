nginx-limit限速

发表于 2018-06-27

一、limit_conn

语法

设置特定键值的共享内存区域和最大允许连接数。超过此限制时，服务器将返回错误以回复请求。

配置


二、limit_conn_zone

语法

key

为共享内存区域设置参数，该区域将保留各种键的状态。特别是，该状态包含当前的连接数。key可以包含文本，变量及其组合。在这里，客户端IP地址作为key。

size

一个1M的区域可以保持大约32000个ipv4的状态或大约16000个ipv6的状态。如果区域存储耗尽，服务器会将错误返回给所有其他请求。


三、limit_rate

语法

  • 零值禁用速率限制

  • 限制是根据请求设置的，所以如果客户端同时打开两个连接，总体速率将是指定限制的两倍

  • 速率限制也可以在$limit_rate变量中设置


四、limit_rate_after

语法

设置初始流量，超过设置的流量后将对客户进一步传送将受到速率限制。

# nginx # limit_conn_zone # limit_conn # limit_rate_after # limit_rate # 限速
